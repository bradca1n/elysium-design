Elysium Design System

Proposed component library reorganization — functional grouping with clear component/widget separation

The Core Principle

Components

Reusable building blocks — product-agnostic, mapped 1:1 to Storybook
  • Button, Input, Select, Checkbox, Radio
  • Modal & Sheet, Toast, Alert, Badge
  • Table, Key-Value Row, Data Field
  • Top Nav Bar, Tabs, Bottom Nav
  • Social Button, OTP Digit, Radio Card

Widgets

Product-specific compositions — built from components, called throughout the product
  • Portfolio Value (Chart + KPI + Values)
  • Fund Holdings (Table + Holding Tiles)
  • Order Summary (Key-Value Rows + Badge)
  • Share Class Card (Data Fields + Badge)
  • Dashboard Summary (KPI Tiles + Charts)

Key Decisions For Discussion

1. Section Grouping

Currently: one section per component. Proposed: group by function (Actions, Data Input, etc.) with multiple components per section.

2. Global vs Platform

Components used across both apps (Button, Input, Modal) stay on Global. Platform-specific (Bottom Nav, Navbar) stay on their app page.

3. Slots as Widgets

Slot components are essentially widgets for Modal/Sheet body content. Keep them grouped by domain (Transactions, Auth, Onboarding).
